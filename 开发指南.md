
# 价格展示与管理系统 - 开发指南

## 开发环境准备

### 硬件要求
- 内存：至少 8GB，建议 16GB
- 磁盘空间：至少 20GB 可用空间
- CPU：Intel i5 或更高

### 软件要求

#### 后端开发环境
- **Java JDK 17+** - 下载地址：https://adoptium.net/
- **Maven 3.6+** - 下载地址：https://maven.apache.org/
- **MySQL 8.0+** - 下载地址：https://www.mysql.com/

#### 前端开发环境
- **Node.js 16+** - 下载地址：https://nodejs.org/
- **Git** - 下载地址：https://git-scm.com/

### IDE推荐
- **后端**: IntelliJ IDEA 或 Eclipse
- **前端**: VS Code 或 WebStorm

## 快速开始

### 1. 克隆项目
```bash
cd E:\ClaudeCodeProject\price-management-system
```

### 2. 数据库初始化

1. 启动MySQL服务
2. 创建数据库：
```sql
CREATE DATABASE IF NOT EXISTS price_management DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE price_management;
```

3. **可选**：导入数据初始化脚本（产品分类和产品数据）：
```sql
-- 执行 backend/src/main/resources/init.sql 文件
source /path/to/init.sql;
```
   - 注意：用户数据由Spring Boot应用启动时的DataInitializer自动初始化，无需手动导入

### 3. 后端项目配置

1. 打开 `backend/src/main/resources/application.yml`
2. 修改数据库连接配置：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/price_management?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: 123abc
    driver-class-name: com.mysql.cj.jdbc.Driver
```

### 4. 后端项目启动

#### 方式一：使用 Maven 命令
```bash
cd backend
mvn clean install
mvn spring-boot:run
```

#### 方式二：使用 IDE
1. 在 IDE 中导入 Maven 项目
2. 找到 `com.pricemanagement.PriceManagementApplication` 类
3. 右键运行

### 5. 前端项目启动

```bash
cd frontend
npm install
npm run dev
```

### 6. 访问应用

- 前端访问：http://localhost:5173
- 后端 API 文档：http://localhost:8080/swagger-ui.html

## 开发流程

### 代码风格规范

#### 后端 Java 代码规范
- 使用 4 个空格缩进
- 变量和方法使用 camelCase
- 类名使用 PascalCase
- 常量使用大写蛇形命名 SNAKE_CASE
- 使用 Lombok 简化代码
- 使用 @Slf4j 进行日志记录

#### 前端代码规范
- 使用 2 个空格缩进
- 变量和方法使用 camelCase
- 组件名使用 PascalCase
- 文件名使用 kebab-case
- 使用 TypeScript 进行类型安全检查
- 使用 ESLint 进行代码检查

### Git 分支管理
- `main`：主分支，用于生产环境
- `develop`：开发分支，用于集成测试
- `feature/*`：功能分支，用于开发新功能
- `hotfix/*`：热修复分支，用于紧急修复

### 提交规范
```
<type>(<scope>): <subject>

<body>

<footer>
```

类型说明：
- `feat`：新功能
- `fix`：修复 bug
- `docs`：文档变更
- `style`：格式变更（不影响代码运行）
- `refactor`：重构（不新增功能，不修复 bug）
- `perf`：性能优化
- `test`：测试相关
- `build`：构建配置变更
- `ci`：CI 配置变更
- `chore`：其他不修改 src 或 test 的变更

## API 接口文档

### 基础信息
- 接口前缀：/api
- 认证方式：Bearer Token (JWT)
- 响应格式：JSON

### 通用响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1704000000000
}
```

### 认证接口

#### 登录
```http
POST /api/auth/login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123"
}
```

#### 获取用户信息
```http
GET /api/auth/profile
Authorization: Bearer {token}
```

### 产品接口

#### 获取产品列表
```http
GET /api/products?page=1&size=10&keyword=硫精砂
Authorization: Bearer {token}
```

#### 获取产品详情
```http
GET /api/products/{id}
Authorization: Bearer {token}
```

#### 创建产品
```http
POST /api/products
Content-Type: application/json
Authorization: Bearer {token}

{
  "name": "新产品",
  "code": "NEW_001",
  "categoryId": 1,
  "status": "ACTIVE"
}
```

### 价格接口

#### 获取产品价格历史
```http
GET /api/products/{id}/price-history
Authorization: Bearer {token}
```

#### 导入产品价格
```http
POST /api/import/products
Content-Type: multipart/form-data
Authorization: Bearer {token}

file: [上传的 Excel 文件]
```

## 常见问题

### 1. 端口被占用
如果 8080 或 5173 端口被占用，可以修改配置：

#### 后端端口修改
修改 `application.yml`：
```yaml
server:
  port: 8081
```

#### 前端端口修改
修改 `vite.config.ts`：
```typescript
server: {
  port: 5174
}
```

### 2. 数据库连接失败
检查以下几点：
- MySQL 服务是否正在运行
- 用户名和密码是否正确
- 数据库名称是否存在
- 防火墙是否允许连接

### 3. 前端无法访问后端 API
检查：
- 后端是否正在运行
- 后端端口是否正确
- CORS 配置是否正确
